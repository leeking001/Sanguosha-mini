## 常见问题排查

### 问题：英雄选择仍然显示 1 列

**可能原因**：
1. 浏览器缓存未清除
2. CSS 文件未正确加载
3. 媒体查询条件不匹配

**解决方案**：
1. 清除浏览器缓存（Ctrl+Shift+Delete）
2. 硬刷新（Ctrl+Shift+R 或 Cmd+Shift+R）
3. 检查 DevTools 中的网络标签，确保 style.css 已加载
4. 检查 DevTools 中的计算样式，验证 grid-template-columns 值

### 问题：血条和技能描述重叠

**可能原因**：
1. 容器宽度不足
2. 元素没有正确设置 flex 属性
3. 文本过长导致溢出

**解决方案**：
1. 检查 #player-status 的宽度（应该是 100% 或接近）
2. 在 DevTools 中检查每个元素的 flex 属性
3. 检查 #my-skill-desc 的 min-width 是否设置为 80px

### 问题：响应式设计在某个尺寸下错乱

**可能原因**：
1. 媒体查询优先级冲突
2. 媒体查询条件重叠
3. 元素尺寸计算错误

**解决方案**：
1. 在 DevTools 中使用 "Show media queries" 功能
2. 检查哪个媒体查询被应用
3. 查看 style.css 中的媒体查询顺序
4. 检查是否有 `!important` 标记导致优先级问题

## 性能检查

### 检查是否有性能问题

1. 打开 DevTools → Performance 标签
2. 点击 Record 按钮
3. 在页面上进行操作（选择英雄、查看血条等）
4. 停止录制
5. 检查帧率是否保持在 60fps

**预期结果**：
- [ ] 帧率保持 60fps
- [ ] 没有长任务（> 50ms）
- [ ] 内存使用稳定
