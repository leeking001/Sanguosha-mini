# PC 手机视图模式实现

## 用户反馈
- 手机端访问效果已经很好了
- **PC 浏览器需要改进** - 在 PC 上也模拟手机端的效果来实现

## 实现方案

### ✨ 新增功能

#### 1. **手机视图切换按钮**
- 在顶部栏添加了 **📱** 按钮（位于语言按钮左侧）
- 点击可以切换桌面视图和手机视图
- 按钮在激活时半透明显示，便于识别当前状态

#### 2. **视图模式选项**
- **桌面视图**：全屏显示，利用整个浏览器窗口
- **手机视图**：固定宽度 480px（标准手机宽度），居中显示，带金色边框

#### 3. **CSS 响应式设计**
在 `style.css` 中添加了 `body.mobile-view` 样式类，包含：

```css
body.mobile-view {
    max-width: 480px;           /* 标准手机宽度 */
    margin: 0 auto;             /* 居中显示 */
    height: 100vh;
    border: 2px solid var(--gold);  /* 金色边框标识 */
    box-shadow: 0 0 20px rgba(212, 175, 55, 0.3);  /* 光晕效果 */
}
```

#### 4. **优化的布局尺寸**
当启用手机视图时，自动应用以下优化：

| 元素 | 手机视图优化 |
|------|------------|
| **顶部栏** | 高度 40px（从 45px 缩小） |
| **敌方区域** | 最大高度 25vh |
| **卡牌** | 70x98px（从标准尺寸缩小） |
| **玩家状态** | 高度 45px，紧凑布局 |
| **英雄选择** | 2 列网格（从 3 列变更） |
| **日志文字** | 字体 12px（从 14px 缩小） |
| **按钮** | 字体 14px，紧凑间距 |

#### 5. **状态持久化**
- 使用浏览器 `localStorage` 保存用户的视图偏好
- 刷新页面后自动恢复上次选择的视图模式
- 键值：`sanguosha_mobile_view` (true/false)

#### 6. **JavaScript 实现**
新增 `initMobileViewToggle()` 函数：
- 检查 localStorage 中的设置
- 初始加载时恢复用户的视图偏好
- 监听按钮点击事件切换视图
- 更新按钮视觉反馈（透明度变化）

### 🎯 使用方式

**开启手机视图：**
1. 点击顶部右上方的 **📱** 按钮
2. 游戏界面自动缩放到 480px 宽度，居中显示
3. 设置自动保存

**关闭手机视图：**
1. 再次点击 **📱** 按钮
2. 游戏恢复全屏桌面视图
