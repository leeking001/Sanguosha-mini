## 修复的Bug列表

### 🔴 严重Bug修复

#### Bug 1: 决斗逻辑错误 - 用【闪】防守决斗 ✅已修复
- **文件**: `game.js` - `resolveDuel()` 函数
- **问题**: 原代码中决斗时被攻击方使用【闪】来防守，这是完全错误的
- **修复**:
  - 决斗中双方轮流出【杀】，被挑战方先出
  - 不出【杀】的一方受到伤害
  - 添加龙胆技能支持：赵云可以用【闪】当【杀】使用

#### Bug 2: 无懈可击使用逻辑错误 ✅已修复
- **文件**: `game.js` - `respondAttack()` 函数
- **问题**: 无懈可击被错误地当作防御卡使用，可以抵消【杀】/【万箭】/【南蛮】
- **修复**: 移除了无懈可击在`respondAttack()`中的错误使用
- **说明**: 无懈可击应该用于抵消锦囊牌效果，而非基本牌攻击

#### Bug 3: 桃园结义未实现 ✅已修复
- **文件**: `game.js` - `useCard()` 函数
- **问题**: 牌堆中有桃园结义但没有实现逻辑
- **修复**: 添加了桃园结义的完整实现 - 所有角色回复1点体力

#### Bug 5: 铁索连环伤害传递后未解除连环状态 ✅已修复
- **文件**: `game.js` - `dealDamage()` 函数
- **问题**:
  - 连环伤害传递后目标仍保持连环状态
  - 多个连环目标只有第一个会受到伤害
- **修复**:
  - 受到伤害的连环角色自动解除连环状态
  - 所有连环角色都会受到传递伤害
  - 添加了 `chain_removed` 事件

### 🟠 中等Bug修复

#### Bug 6: 乐不思蜀判定逻辑错误 ✅已修复
- **文件**: `game.js` - `startTurn()` 函数
- **问题**: 原代码使用50%几率判定，应该是红桃(约25%)跳过乐不
- **修复**: 改为25%几率判定成功（红桃跳过），75%几率生效（跳过出牌阶段）

#### Bug 7: AI决斗策略有误 ✅已修复
- **文件**: `ai.js` - `takeTurn()` 函数
- **问题**: AI只判断目标手牌数，没有评估自己的杀数量
- **修复**: AI现在会比较自己的杀数量和对方估计的杀数量，只有优势时才发起决斗

#### Bug 8: 龙胆技能在决斗中不生效 ✅已修复
- **文件**: `game.js` - `resolveDuel()` 函数
- **问题**: 赵云的龙胆技能在决斗中无法使用【闪】当【杀】
- **修复**: 在决斗逻辑中添加了龙胆技能检查

#### Bug 10: 内奸AI逻辑错误 ✅已修复
- **文件**: `ai.js` - `getEnemyTarget()` 函数
- **问题**: 内奸优先攻击忠臣，这是错误的策略
- **修复**: 内奸早期优先攻击反贼（帮助主公），后期才攻击忠臣

## 技术改进

1. **决斗机制重写**: 完全重写了决斗逻辑，现在正确实现了被挑战方先出、双方轮流出杀的规则

2. **连环伤害传递**: 修复了连环伤害只传一人的bug，现在会传递给所有连环角色

3. **事件系统增强**: 添加了 `chain_removed`、`judgment_fail`、`judgment_success` 等新事件类型

4. **AI策略优化**:
   - 内奸策略从"打忠臣"改为"早期打反贼，后期打忠臣"
   - 决斗前会评估双方杀数量
