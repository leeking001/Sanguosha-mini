📋 Test Checklist

1. Mobile View Button Visibility PENDING

Check if the 📱 button is visible in the top-right corner of the game interface

2. Toggle Functionality PENDING

Click the 📱 button and verify:

  • Game container becomes 480px wide
  • Game is centered on screen
  • Gold border appears around the game
  • Button opacity changes to 0.7 (semi-transparent)
3. Desktop View Recovery PENDING

Click the 📱 button again and verify:

  • Game returns to full-screen display
  • Gold border disappears
  • Button opacity returns to 1.0 (fully opaque)
4. UI Element Sizing PENDING

In mobile view, verify:

  • Cards appear smaller (70x98px)
  • Text is appropriately scaled down
  • Buttons and controls are properly sized
  • Hero selection shows 2 columns instead of 3
5. localStorage Persistence PENDING

Test persistence:

  1. Enable mobile view (click 📱 button)
  2. Refresh the page (F5 or Cmd+R)
  3. Verify mobile view is still active after refresh
  4. Check browser console: localStorage.getItem('sanguosha_mobile_view') should return 'true'
6. Game Functionality Unaffected PENDING

In both views, verify:

  • Game buttons are clickable
  • No visual glitches or overlapping elements
  • All game features work normally

🔍 How to Run Tests

Step 1: Open the Game

Step 2: Browser Console Commands
Use these commands in the browser console to test programmatically:
// Check current state document.body.classList.contains('mobile-view')

// Get stored preference localStorage.getItem('sanguosha_mobile_view')

// Manually toggle mobile view document.body.classList.toggle('mobile-view')

// Set mobile view document.body.classList.add('mobile-view')

// Remove mobile view document.body.classList.remove('mobile-view')

// Check CSS applied window.getComputedStyle(document.body).maxWidth
Step 3: Check DevTools
In the Elements tab:
  • When mobile view is active, <body> should have class "mobile-view"
  • Applied styles should show: max-width: 480px, margin: 0 auto, border: 2px solid gold
